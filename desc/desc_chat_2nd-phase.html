[A] 사이드바 - 채팅 관리

2차에서는 단발성 대화에서 벗어나 대화 히스토리를 기록·관리할 수 있도록 개선되었다. 사이드바에서 트리거되는 삭제 확인 모달과 채팅 검색 팝업도 본 섹션에 함께 기술한다.

No.항목설명
1 채팅 목록 영역신규 사이드바에 사용자의 대화 히스토리를 노출하는 영역
  • 위치: 사이드바 '새 채팅', '검색' 버튼 하단
  • 구성: 고정 영역 + 구분선 + 일반 영역
  • 고정 영역과 일반 영역이 모두 존재할 때만 둘 사이에 구분선 노출
  • 정렬: 고정 영역 / 일반 영역 모두 최근 생성 순(내림차순)
  • 대화 항목이 많을 경우 세로 스크롤 처리
  • 클릭 시 해당 대화 화면으로 진입
2 대화 항목 호버 동작신규 대화 항목 위에 마우스를 올렸을 때만 메뉴 버튼(...) 노출
  • 디폴트: 미호버 시 메뉴 버튼 숨김 (제목만 노출)
  • 호버 시 우측 끝에 더보기 아이콘 노출
  • 더보기 클릭 시 컨텍스트 메뉴 펼침 → No.3 참고
3 채팅 컨텍스트 메뉴신규 대화 항목 더보기(...) 클릭 시 노출되는 메뉴
  • 구성: 고정 / 이름 변경 / 삭제 3개 항목
  • 일반 영역의 대화: '고정'으로 노출
  • 고정 영역의 대화: '고정 해제'로 노출 (동일 메뉴 위치)
  • '삭제' 항목은 시각적 강조를 위해 빨간색 텍스트로 표시
  • 메뉴 외부 클릭 시 메뉴 자동 닫힘
4 채팅 고정신규 자주 사용하는 대화를 사이드바 상단으로 고정하는 기능
  • 동작: 컨텍스트 메뉴(No.3)의 '고정' 클릭 시 일반 영역 → 고정 영역으로 이동
  • 고정 영역 대화 좌측에 핀 아이콘(메인 컬러) 노출
  • '고정 해제' 클릭 시 일반 영역으로 복귀
  • 고정 가능 개수 제한: 별도 제한 없음 (정책서에 별도 정의 없음)
5 대화 이름 변경신규 대화 제목을 사용자가 직접 수정할 수 있는 기능
  • 진입: 컨텍스트 메뉴(No.3)의 '이름 변경' 클릭
  • 해당 대화 항목이 인라인 입력창으로 전환되며 자동 포커스
  • 입력 제한: 최대 15자까지 입력 가능 (15자 초과 입력 시 자동 절단)
  • 저장 트리거:
    • Enter 키 입력 시 저장
    • 입력창 외부 영역 클릭(blur) 시 저장
  • 취소: Esc 키 입력 시 변경 사항 무시하고 원본 유지
  • 빈 값 저장 시도 시: 변경 무시 (원본 제목 유지)
6 대화 삭제신규 대화 히스토리를 영구 삭제하는 기능
  • 진입: 컨텍스트 메뉴(No.3)의 '삭제' 클릭
  • 즉시 삭제하지 않고 삭제 확인 모달을 먼저 노출 → No.7 참고
  • 모달의 '삭제' 확정 시 채팅 목록에서 제거
  • 삭제된 대화는 복구 불가
7 삭제 확인 모달신규팝업 대화 삭제(No.6) 실행 전 한 번 더 확정을 받기 위한 확인 모달
  • 진입: 컨텍스트 메뉴(No.3)의 '삭제' 클릭
  • 구성:
    • 제목: "대화를 삭제하시겠어요?"
    • 설명: "삭제된 대화는 복구할 수 없습니다."
    • 버튼: 취소(일반 보더) / 삭제(빨간색, 위험 강조)
  • '삭제' 클릭 시: 대화 항목을 채팅 목록에서 영구 삭제 후 모달 닫힘
  • '취소' 클릭 시: 변경 없이 모달만 닫힘
  • 현재는 모달 외부 영역 클릭으로 닫히지 않음 (실수 방지를 위해 명시적 버튼 클릭 필요)
8 채팅 제목 자동 생성신규 새 대화에서 첫 메시지를 전송하면 시스템이 대화 제목을 자동으로 생성
  • 트리거: 새 채팅(메시지 0건 상태)에서 사용자가 첫 메시지 전송 시
  • 제목 규칙: 첫 메시지 텍스트 기준 최대 15자까지 사용
  • 15자 초과 시: 15자 + 말줄임표(…) 형태로 표시 (예: "지난 한달 간 PC필터 판매…")
  • 자동 생성된 제목은 채팅 목록 일반 영역 최상단에 신규 추가
  • 이후 사용자가 '이름 변경'(No.5) 으로 직접 수정 가능
9 채팅 검색 팝업신규팝업 대화 히스토리 및 현재 대화 내 메시지를 검색할 수 있는 팝업
  • 진입: 사이드바 '검색' 버튼 클릭
  • 위치: 화면 중앙 상단(top 10vh)에 모달 형태로 노출
  • 구성:
    • 상단: 돋보기 + 검색어 입력창 (자동 포커스) + 닫기(X) 버튼
    • 하단: 검색 결과 목록
  • 검색 동작: 입력 즉시 실시간 검색 (트리거 클릭 불필요)
  • 검색 대상:
    • 대화 목록 제목
    • 현재 대화의 메시지(사용자 입력 + AI 답변)
  • 결과 항목 구성:
    • 대화 제목 결과: 제목 텍스트만 표시
    • 메시지 결과: '내 메시지' / 'AI 큐피드' 라벨 + 메시지 본문 1줄 (line-clamp)
  • 결과 클릭 시: 해당 대화 화면으로 진입 후 팝업 자동 닫힘
  • 검색어 미입력 상태: "검색어를 입력하세요." 안내
  • 검색 결과 없음: "'{검색어}'에 대한 결과가 없습니다." 안내
  • 닫기:
    • 우측 상단 X 버튼 클릭
    • 모달 외부(어두운 배경) 영역 클릭

[B] 응답 만족도 피드백

AI 답변 하단에 만족도 평가 영역을 노출하여 피드백을 수집한다. (정책서 5장 참고) 싫어요 클릭 시 노출되는 사유 입력 팝업도 본 섹션에 함께 기술한다.

No.항목설명
10 피드백 영역 노출신규 AI가 답변을 제공한 모든 메시지 하단에 만족도 평가 영역을 상시 노출
  • 위치: 각 AI 답변 메시지의 하단
  • 구성: 안내 문구 "답변이 도움이 됐나요?" + 좋아요/싫어요 아이콘
  • 노출 조건: AI 답변이 출력된 모든 메시지에 항상 노출 (조건부 노출 없음)
  • 각 답변마다 독립적으로 피드백 상태 관리 (메시지별로 별도 저장)
11 좋아요(👍)신규 답변에 만족한 경우 선택하는 긍정 피드백 버튼
  • 디폴트 상태: 비활성 (회색)
  • 클릭 시: 즉시 저장 + 활성화(메인 컬러로 변경)
  • 재 클릭 시: 좋아요 취소 (비활성 상태로 복귀)
  • 좋아요 ↔ 싫어요는 상호 배타: 좋아요 클릭 시 기존에 선택된 싫어요는 자동 해제
12 싫어요(👎)신규 답변에 불만족한 경우 선택하는 부정 피드백 버튼
  • 디폴트 상태: 비활성 (회색)
  • 클릭 시: 즉시 싫어요 사유 입력 팝업 노출 → No.13 참고
  • 사유까지 선택 완료해야 싫어요 상태가 정식 반영됨 (활성화/저장)
  • 재 클릭 시: 싫어요 취소 (비활성 상태로 복귀, 저장된 사유도 함께 초기화)
  • 좋아요 ↔ 싫어요는 상호 배타: 싫어요 클릭 시 기존에 선택된 좋아요는 자동 해제
13 싫어요 사유 입력 팝업신규팝업 싫어요(No.12) 클릭 시 사유를 수집하기 위해 자동으로 노출되는 팝업
  • 진입: 답변 하단 싫어요(👎) 아이콘 클릭
  • 위치: 화면 정중앙 모달
  • 구성:
    • 제목: "어떤 점이 불편하셨나요?" + 닫기(X) 버튼
    • 안내: "복수 선택 가능"
    • 사유 체크박스 5종 (정책서 5-2 기준)
    • '기타' 선택 시: 직접 입력 텍스트박스 인라인 노출
    • 하단: 확인 버튼 (메인 컬러)
  • 사유 체크박스 항목:
    1. 결과 데이터가 부정확해요
    2. 질문 의도와 다른 답변이에요
    3. 답변 설명이 부족하거나 이해하기 어려워요
    4. 시각화(표/차트)가 부적절해요
    5. 기타 (직접 입력)
  • '확인' 버튼 동작:
    • 최소 1개 이상 사유 선택 필수
    • 사유 선택 완료(또는 '기타' + 텍스트 입력) 후 확인 클릭 시 → 싫어요 상태 정식 반영 + 팝업 닫힘
    • 사유 미선택 상태에서 확인 클릭 시 → 안내 메시지 노출 (예: "사유를 1개 이상 선택해주세요.")
      ※ 안내 메시지 형태(인라인/토스트)는 별도 협의 필요
    • '기타'만 선택하고 텍스트 미입력 상태: 사유 미선택과 동일하게 처리 (안내 메시지)
  • 'X' 버튼 또는 모달 외부 영역 클릭 시: 싫어요 자체가 취소 (저장된 사유도 함께 초기화)

[C] 데이터 테이블 가공

1차에서는 고정 형태로 제공되던 결과 테이블을 2차에서는 사용자가 정렬·필터·초기화할 수 있도록 가공 기능을 추가한다.

No.항목설명
14 키워드 필터신규 테이블 데이터에서 키워드를 포함한 행만 필터링하는 기능
  • 위치: 테이블 상단 툴바 좌측
  • 입력 방식: 텍스트 입력 후 돋보기 아이콘 클릭 또는 Enter 키로 적용
  • 적용 규칙: 입력한 키워드를 모든 컬럼에서 검색하여 일치하는 행만 표시 (대소문자 구분 없음)
  • 필터 적용 시 페이지네이션은 1페이지로 자동 이동
  • 일치하는 행이 없는 경우: "검색 결과가 없어요." 안내 문구 노출
  • 검색 입력 즉시 자동 필터링이 아닌, 명시적 트리거 후 적용 (사용자가 의도한 시점에 적용)
15 컬럼 정렬신규 테이블 헤더 클릭으로 해당 컬럼 기준 오름차순/내림차순 정렬
  • 디폴트 상태: 정렬 미적용 (각 컬럼 헤더에 ↕ 아이콘 노출)
  • 최초 클릭: 오름차순 정렬 (▲ 아이콘으로 변경)
  • 동일 컬럼 재 클릭: 내림차순 정렬 (▼ 아이콘으로 변경)
  • 다른 컬럼 클릭: 새 컬럼 기준 오름차순으로 정렬 (이전 정렬 해제)
  • 한 번에 한 컬럼 기준으로만 정렬 (다중 컬럼 정렬 미지원)
16 테이블 초기화신규 테이블에 적용된 가공(정렬·필터)을 한 번에 디폴트 상태로 되돌리는 기능
  • 위치: 테이블 상단 툴바 (키워드 필터 우측)
  • 아이콘: 되돌리기(↻) 아이콘
  • 리셋 범위: 해당 테이블 영역에 적용된 가공만 초기화
    • 키워드 필터 → 빈 값으로 초기화
    • 컬럼 정렬 → 정렬 미적용 상태로 초기화
    • 페이지네이션 → 1페이지로 이동
  • 차트 영역(No.19)의 가공 상태에는 영향을 주지 않음

[D] 통계 표 / 차트 가공

차트 답변 영역에 통계 표 정렬 및 차트 유형 전환 기능을 추가한다.

No.항목설명
17 통계 표 컬럼 정렬신규 차트 답변에 함께 노출되는 통계 표의 컬럼 정렬 기능
  • 적용 컬럼: 순위 / 제품명 / 판매 건수 / 비중(%)
  • 동작 방식: 데이터 테이블 정렬(No.15)과 동일한 토글 방식 (↕ → ▲ → ▼)
  • 한 번에 한 컬럼 기준으로만 정렬
18 차트 유형 전환변경 제공된 차트의 시각화 유형을 사용자가 직접 변경할 수 있는 기능
  • 위치: 차트 영역 헤더 우측
  • 선택 가능 유형: 4종으로 한정
    • 원형 (Pie)
    • 세로 막대 (Vertical Bar)
