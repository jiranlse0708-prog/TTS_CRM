[A] 계산식 - 검색 / 추가

No.항목설명
1 계산식 검색 입력창신규 등록된 계산식을 키워드로 필터링하는 검색창
  • 위치: 테이블 상단 좌측
  • 플레이스홀더: "계산식 검색"
  • 검색 동작: 입력 즉시 실시간 필터링
  • 검색 대상 컬럼: 지표명 / 계산식 2개 모두 (부분 일치)
  • 탭/뷰 전환 시 검색어 자동 초기화
2 '+ 추가' 버튼신규관리자만 신규 계산식을 등록하기 위한 추가 버튼
  • 노출 조건: 관리자 권한 계정에서만 노출
  • 위치: 검색 입력창 우측
  • 버튼 스타일: 메인 컬러 배경 + '+' 아이콘 + '추가' 텍스트
  • 클릭 시 동작:
    • 테이블 최상단에 인라인 추가 행 노출 (강조 배경색)
    • 편집 중인 행이 있다면 자동 종료
    • '지표명' 입력 필드에 자동 포커스
3 인라인 추가 행 (입력)신규관리자만 '추가' 클릭 시 테이블 최상단에 노출되는 신규 등록 행
  • 위치: 테이블 최상단 (헤더 바로 아래)
  • 입력 필드 (정책서 4-1 기준):
    • 지표명 (필수, placeholder: "지표명 *")
    • 계산식 (필수, placeholder: "계산식 *")
  • 저장 버튼 (✓ 체크):
    • 지표명·계산식 모두 입력된 경우에만 등록 처리
    • 둘 중 하나라도 공란이면 저장되지 않음 (별도 에러 메시지 노출 여부 확인 필요)
    • 저장 성공 시: 신규 행으로 목록에 추가 + 추가 행 닫힘
  • 취소 버튼 (X): 입력값 무시하고 추가 행 닫힘
  • 입력 길이 제한: 알 수 없음 (코드·정책서 모두 미정의)
  • 계산식 표기 규칙: 정책서 예시는 자유 텍스트 형식 (예: "현월 매출 − 전월 매출", "(현월 값 − 전월 값) / 전월 값 × 100") — 구문 검증·연산자 제약 등은 정의되지 않음

[B] 계산식 - 목록 (테이블)

No.항목설명
4 계산식 목록 테이블신규 등록된 계산식을 표 형태로 노출하는 영역
  • 컬럼 구성:
    • 지표명 (너비 200px)
    • 계산식 (가변 너비)
    • 작업 (너비 72px) — 관리자만 노출
  • 일반 사용자: '작업' 컬럼이 헤더·행 모두 미노출 (2개 컬럼만 표시)
  • 행 정렬: 알 수 없음 (등록 순으로 추정, 정책서 미정의)
  • 줄무늬: 짝수 행에 옅은 회색 배경
  • 페이지네이션: 없음 (단일 스크롤, 전체 노출)
5 빈 상태 / 검색 결과 없음신규 목록이 비어 있거나 검색 결과가 없을 때 안내 문구
  • 안내 행: 테이블 본문 영역 가운데에 텍스트 1줄 노출
  • 케이스 분기:
    • 등록된 계산식이 없는 경우: "등록된 계산식이 없어요."
    • 검색어가 있고 결과가 없는 경우: "검색 결과가 없어요."
  • 추가 행이 열려 있는 동안에는 빈 상태 메시지가 노출되지 않음

[C] 계산식 - 행 편집/삭제

No.항목설명
6 행 작업 아이콘 (수정/삭제)신규관리자만 행별로 노출되는 수정/삭제 액션 아이콘
  • 노출 조건: 관리자 권한 계정에서만 '작업' 컬럼에 아이콘 노출
  • 구성: 연필 아이콘(수정) + 휴지통 아이콘(삭제) - 가운데 정렬
  • 호버 시:
    • 수정 아이콘: 메인 컬러 + 옅은 보라색 배경
    • 삭제 아이콘: 빨간색 + 옅은 빨간색 배경
  • 일반 사용자에게는 본 컬럼 자체가 미노출
7 인라인 수정 모드신규관리자만 행의 연필 아이콘 클릭 시 해당 행이 입력 모드로 전환
  • 진입: 행의 연필(수정) 아이콘 클릭
  • 전환 동작:
    • 해당 행의 2개 셀(지표명·계산식) 이 입력 필드로 변경되고 기존 값이 채워짐
    • 강조 배경색(연한 보라색)으로 편집 중 상태 시각화
    • '추가' 모드가 열려 있었다면 자동 닫힘 (수정과 추가는 동시 진행 불가)
  • 저장 버튼 (✓): 입력값으로 행 갱신 + 편집 모드 종료
  • 취소 버튼 (X): 변경 무시 + 편집 모드 종료 (원본 유지)
  • 다른 행을 동시에 편집할 수 없음 (한 번에 한 행만)
  • 저장 시 필수값 검증: 알 수 없음 (수정에는 빈 값 검증이 없는 것으로 보임)
8 행 삭제신규관리자만 행의 휴지통 아이콘 클릭 시 해당 계산식 삭제
  • 진입: 행의 휴지통 아이콘 클릭
  • 동작: 확인 모달 없이 즉시 삭제 (코드 기준)
    • ※ 채팅 삭제와 달리 사전 확인 모달 없음 → 정책 확인 필요
  • 삭제 후 목록 즉시 갱신
  • 삭제된 계산식은 복구 불가
