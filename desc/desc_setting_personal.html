본 화면은 좌측 서브 메뉴에서 '개인 맞춤 설정'을 선택했을 때 노출됩니다. 진입 경로 / 좌측 서브 메뉴는 용어 정의 디스크립션(desc_setting_term.html) [A] 섹션을 참고하세요. 본 문서는 '개인 맞춤 설정' 화면 고유 영역만 기술합니다.
※ 정책서 4-2(답변 스타일 설정) 기준 — 본 설정은 본인 계정에만 적용되며, 관리자도 타 계정의 설정을 조회·수정할 수 없습니다.

[A] 페이지 헤더

No.항목설명
1 페이지 헤더신규 '개인 맞춤 설정' 타이틀과 설명 영역
  • 구성: 타이틀 "개인 맞춤 설정"
    • '관리자 전용' 뱃지는 노출되지 않음 (본인 계정 전용 설정이므로 권한 분기 없음)
  • 안내 문구: "개인이 선호하는 답변 스타일을 설정합니다. 큐피드AI의 기본 원칙 내에서 본인 계정의 모든 대화에 반영됩니다."
  • 적용 범위: 본인 계정에만 적용 (정책서 4-2)
  • 편집 권한: 본인만 등록·수정·삭제 가능
  • 관리자도 타 계정의 본 설정을 조회·수정 불가

[B] 답변 스타일 입력 영역

No.항목설명
2 답변 스타일 텍스트 입력신규 개인 답변 스타일을 자유 텍스트로 작성하는 멀티라인 입력 영역
  • 위치: 페이지 헤더 하단 콘텐츠 영역 (최대 너비 600px)
  • 입력 방식: 멀티라인 텍스트박스 (textarea, 높이 220px 고정, resize 비활성)
  • 입력 길이 제한: 최대 2,000자 (정책서 4-2 기준)
    • 2,000자 초과 입력 시 추가 입력이 차단됨 (입력 자체가 막힘)
  • 이력 관리: 최신 본만 유지 (이력 저장하지 않음, 정책서 4-2)
  • 플레이스홀더 (예시 4줄): 
    예시)
- 모든 금액은 백만 원 단위로 표시해주세요.
- 소수점은 두 번째 자리에서 반올림해주세요.
- 날짜는 YYYY년 M월 형식으로 표기합니다.
- 10줄 이내 분량으로 자세하게 풀어서 설명해주세요.
  • 입력값 변경 시 '저장됨' 상태 자동 해제 → No.4 참고
  • 수용 가능 항목 (정책서 4-2 기준):
    • 표기 형식 (금액 단위, 소수점, 날짜 형식 등)
    • 말투 / 어조 / 분량
  • 수용 불가 항목: 위 카테고리 외 입력 시 LLM이 무시 (정책서 명시)
  • ※ 답변 구성 요소(표·차트·SQL 등)의 구성 자체는 본 설정으로 변경 불가능 (정책서 4-2)
3 글자 수 카운터신규 현재 입력된 글자 수와 최대 글자 수를 함께 표시
  • 위치: 텍스트박스 하단 좌측
  • 표기 형식: "{현재}/2,000자" (예: "327 / 2,000자")
  • 현재 글자 수에는 천 단위 콤마 표기 적용
  • 색상 변화: 별도 강조 색상 분기 없음 (회색 텍스트 고정)

[C] 저장 / 안내

No.항목설명
4 저장 버튼 (상태 변화)신규 입력한 답변 스타일을 저장하는 버튼 + 저장 직후 피드백
  • 위치: 텍스트박스 하단 우측 (글자 수 카운터 반대편)
  • 디폴트 상태: 메인 컬러 배경 + '저장' 텍스트 (활성)
  • 클릭 시 동작:
    • 입력값을 즉시 저장 처리
    • 버튼이 '저장됨 ✓' (연한 초록 배경 + 진한 초록 텍스트) 로 시각 피드백 전환
    • 2초 후 자동으로 '저장' 상태로 복귀
  • 입력값을 다시 변경하면 '저장됨' 상태가 즉시 해제되고 일반 '저장' 버튼으로 돌아감
  • 빈 값 저장 처리: 알 수 없음 — 코드상 글자 수 0이어도 저장 가능 (의도적 초기화 허용 여부 정책 확인 필요)
  • 저장 실패 케이스 (서버 오류 등) 의 안내 처리: 알 수 없음 (정책서·코드 미정의)
5 등록 예시 안내 박스신규 사용자가 어떤 항목을 입력하면 좋은지 안내하는 정보 박스
  • 위치: 저장 버튼 하단
  • 구성:
    • 제목: "등록 예시 항목"
    • 본문: "금액 단위, 소수점 반올림, 날짜 표기 형식, 답변 톤·어조 등을 자유롭게 입력하세요."
  • 스타일: 옅은 회색 배경의 정보 박스 (인터랙션 없음, 정적 안내)
  • 사용자가 어떤 카테고리의 입력이 수용 가능한지 가이드 제공
6 우선순위 적용 규칙 (참고)신규 설정 유무에 따른 답변 스타일 적용 우선순위 (정책서 4-2 기준, 화면 미노출)
  • ※ 화면에 직접 노출되는 항목은 아니지만, 동작 정책상 명시 필요
  • 설정값이 비어 있을 때: 기본 페르소나 동작 적용 (CRM 1차 정책서 4-1 참고)
  • 설정값이 있을 때: 입력된 답변 스타일을 우선 적용
  • 본 설정은 답변 생성 단계에서 LLM에 전달됨 (쿼리문 생성 단계 X)
