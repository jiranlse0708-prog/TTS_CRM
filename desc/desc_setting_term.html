[A] 진입 경로 / 좌측 서브 메뉴

맞춤 설정 화면의 공통 영역(사이드바 진입 + 좌측 2뎁스 메뉴)을 정의한다. 본 영역은 모든 맞춤 설정 화면에서 동일하게 노출된다.

No.항목설명
1 '맞춤 설정' 진입 버튼신규 사이드바 하단에 위치한 맞춤 설정 진입 버튼
  • 위치: 사이드바 최하단 (사용자 프로필 영역 바로 위)
  • 아이콘 + '맞춤 설정' 텍스트 라벨
  • 현재 뷰가 맞춤 설정인 경우: 메인 컬러 배경 + 보라색 텍스트로 활성 상태 표시
  • 채팅 뷰에서 클릭 시: 맞춤 설정 뷰로 전환 + 디폴트 탭(비즈니스 공통 설정 > 용어 정의)으로 진입
  • 맞춤 설정 뷰에서 재 클릭 시: 채팅 뷰로 복귀 (토글)
  • 맞춤 설정 진입 시 검색어·추가 폼·편집 상태는 모두 초기화됨
2 좌측 서브 메뉴 (2뎁스)신규 맞춤 설정 뷰의 좌측에 노출되는 카테고리 메뉴
  • 위치: 사이드바 우측, 콘텐츠 영역 좌측 (너비 168px 고정)
  • 헤더 라벨: "설정"
  • 구성 (정책서 4장 기준 2개 카테고리):
    1. 비즈니스 공통 설정 (= 정책서의 '비즈니스 맞춤 설정')
    2. 개인 맞춤 설정 (= 정책서의 '답변 스타일 설정')
  • 활성 상태 표시: 현재 선택된 메뉴는 메인 컬러 배경 + 보라색 텍스트 (Bold)
  • '비즈니스 공통 설정' 클릭 시: 검색어·추가 폼·편집 상태 모두 초기화
  • 디폴트 진입 메뉴: 비즈니스 공통 설정

[B] 비즈니스 공통 설정 - 헤더

'비즈니스 공통 설정' 메뉴 진입 시 콘텐츠 상단에 노출되는 페이지 헤더 영역이다. 용어 정의·계산식 두 서브탭이 공유한다.

No.항목설명
3 페이지 헤더 + 권한 뱃지신규 '비즈니스 공통 설정' 타이틀과 권한 안내
  • 구성: 타이틀 "비즈니스 공통 설정" + '관리자 전용' 뱃지
  • 안내 문구: "사내 기준을 설정하면 비즈니스 문맥을 이해하여 더 정확한 답변을 제공합니다. 등록된 설정은 모든 사용자에게 동일하게 적용됩니다."
  • 적용 범위: 본 설정은 모든 사용자에게 동일 적용됨 (정책서 4-1 기준)
4 일반 사용자 안내 박스신규일반 사용자 관리자 권한이 없는 사용자에게만 노출되는 조회 전용 안내
  • 노출 조건: 일반 사용자(비관리자) 권한 계정으로 진입한 경우에만 노출
  • 위치: 헤더 안내 문구 하단
  • 안내 문구: "조회만 가능합니다. 등록·수정·삭제는 관리자 권한이 필요해요."
  • 관리자 권한 계정에서는 본 박스 자체가 노출되지 않음
5 서브탭: 용어 정의 / 계산식신규 비즈니스 공통 설정 내 2개 서브 탭
  • 구성: 용어 정의 / 계산식 2개 탭
  • 디폴트: 용어 정의 탭
  • 활성 탭: 메인 컬러 하단 라인 + 보라색 텍스트(Bold) 로 강조
  • 탭 전환 시: 검색어·추가 폼·편집 상태 모두 초기화 (탭 간 입력 상태가 섞이지 않도록)
  • '계산식' 탭에 대한 상세는 별도 디스크립션(desc_setting_formula.html) 참고

[C] 용어 정의 - 검색 / 추가

No.항목설명
6 용어 검색 입력창신규 등록된 용어를 키워드로 필터링하는 검색창
  • 위치: 테이블 상단 좌측
  • 플레이스홀더: "용어 검색"
  • 검색 동작: 입력 즉시 실시간 필터링 (트리거 클릭 불필요)
  • 검색 대상 컬럼: 용어명 / 정의 / 동의어·약어 3개 모두 (부분 일치 검색)
  • 대소문자 구분: 별도 처리 없음 (입력 그대로 일치 검색)
  • 탭 전환·뷰 전환 시 검색어 자동 초기화 → No.5, No.1 참고
7 '+ 추가' 버튼신규관리자만 신규 용어를 등록하기 위한 추가 버튼
  • 노출 조건: 관리자 권한 계정에서만 노출 (일반 사용자는 버튼 자체 미노출)
  • 위치: 검색 입력창 우측
  • 버튼 스타일: 메인 컬러 배경 + '+' 아이콘 + '추가' 텍스트
  • 클릭 시 동작:
    • 테이블 최상단에 인라인 추가 행이 노출됨 (강조 배경색)
    • 편집 중인 행이 있다면 자동 종료 (편집 모드 해제)
    • 입력 필드 3개 (용어명·정의·동의어·약어) 가 자동 초기화되어 노출
    • '용어명' 입력 필드에 자동 포커스 (autoFocus)
8 인라인 추가 행 (입력)신규관리자만 '추가' 클릭 시 테이블 최상단에 노출되는 신규 등록 행
  • 위치: 테이블 최상단 (헤더 바로 아래)
  • 입력 필드 (정책서 4-1 기준):
    • 용어명 (필수, placeholder: "용어명 *")
    • 정의 (필수, placeholder: "정의 *")
    • 동의어·약어 (선택, placeholder: "동의어·약어")
  • 저장 버튼 (✓ 체크 아이콘):
    • 용어명·정의 모두 입력된 경우에만 등록 처리
    • 둘 중 하나라도 공란이면 저장되지 않음 (별도 에러 메시지 노출 여부 확인 필요 - 정책서에 별도 정의 없음)
    • 저장 성공 시: 신규 행으로 목록에 추가 + 추가 행 닫힘
  • 취소 버튼 (X 아이콘): 입력값 무시하고 추가 행 닫힘
  • 입력 길이 제한: 알 수 없음 (코드·정책서 모두 별도 제한 미정의)

[D] 용어 정의 - 목록 (테이블)

No.항목설명
9 용어 목록 테이블신규 등록된 용어를 표 형태로 노출하는 영역
  • 컬럼 구성:
    • 용어명 (너비 150px)
    • 정의 (가변 너비)
    • 동의어·약어 (너비 150px)
    • 작업 (너비 72px) — 관리자만 노출
  • 일반 사용자: '작업' 컬럼이 헤더·행 모두 미노출 (3개 컬럼만 표시)
  • 행 정렬: 알 수 없음 — 등록 순서 또는 등록일시 기준 추정 (정책서 미정의, 코드상 등록 순으로 push)
  • 줄무늬: 짝수 행에 옅은 회색 배경 적용 (가독성)
  • 페이지네이션: 없음 (단일 스크롤, 전체 노출)
10 빈 상태 / 검색 결과 없음신규 목록이 비어 있거나 검색 결과가 없을 때 안내 문구
  • 안내 행: 테이블 본문 영역 가운데에 텍스트 1줄 노출
  • 케이스 분기:
    • 등록된 용어가 없는 경우 (검색어도 없음): "등록된 용어가 없어요."
    • 검색어가 있고 결과가 없는 경우: "검색 결과가 없어요."
  • 추가 행이 열려 있는 동안에는 빈 상태 메시지가 노출되지 않음 (입력 중에 메시지가 끼어들지 않도록)

[E] 용어 정의 - 행 편집/삭제

No.항목설명
11 행 작업 아이콘 (수정/삭제)신규관리자만 행별로 노출되는 수정/삭제 액션 아이콘
  • 노출 조건: 관리자 권한 계정에서만 '작업' 컬럼에 아이콘 노출
  • 구성: 연필 아이콘(수정) + 휴지통 아이콘(삭제) - 가운데 정렬
  • 호버 시:
    • 수정 아이콘: 메인 컬러로 변경 + 옅은 보라색 배경
    • 삭제 아이콘: 빨간색으로 변경 + 옅은 빨간색 배경
  • 일반 사용자에게는 본 컬럼 자체가 노출되지 않음
12 인라인 수정 모드신규관리자만 행의 연필 아이콘 클릭 시 해당 행이 입력 모드로 전환
  • 진입: 행의 연필(수정) 아이콘 클릭
  • 전환 동작:
    • 해당 행의 3개 셀이 입력 필드로 변경되고 기존 값이 그대로 채워짐
    • 강조 배경색(연한 보라색)으로 편집 중 상태 시각화
    • '추가' 모드가 열려 있었다면 자동 닫힘 (수정과 추가는 동시 진행 불가)
  • 저장 버튼 (✓): 입력값으로 해당 행 갱신 + 편집 모드 종료
  • 취소 버튼 (X): 변경 무시 + 편집 모드 종료 (원본 유지)
  • 다른 행을 동시에 편집할 수 없음 (한 번에 한 행만 편집 가능)
  • 저장 시 필수값 검증: 알 수 없음 — 코드상 추가와 달리 수정에는 빈 값 검증이 없는 것으로 보임 (정책 확인 필요)
13 행 삭제신규관리자만 행의 휴지통 아이콘 클릭 시 해당 용어 삭제
  • 진입: 행의 휴지통(삭제) 아이콘 클릭
  • 동작: 확인 모달 없이 즉시 삭제 (코드 기준)
    • ※ 채팅 삭제와 달리 사전 확인 모달이 없음 → 정책 확인 필요 (오삭제 방지 차원에서 확인 모달 추가 여부 검토 권장)
  • 삭제 후 목록이 즉시 갱신됨
  • 삭제된 용어는 복구 불가
